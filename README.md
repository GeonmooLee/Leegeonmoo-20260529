# 살뜰: AI 쇼핑 실행 도우미

살뜰은 가족 일정과 생활 쇼핑이 뒤섞인 메모에서 구매할 물품을 정리하고, 필요한 시점에 상품 후보를 찾도록 돕는 풀스택 MVP입니다. 일반 일정 관리가 아니라 집안 장보기의 기억 부담을 줄이는 실행 도구를 목표로 합니다.

## 주요 흐름

1. 사용자가 메모 탭에 자유로운 가족 메모를 입력합니다.
2. AI가 일정별 물품을 `사용자 언급 물품`과 `AI 추가 추천 물품`으로 분리합니다.
3. 사용자는 제안 내용을 수정하거나 직접 항목을 보완한 뒤 구매 리스트에 담습니다.
4. 구매 리스트에서 마감일, 우선순위, 비고를 관리하고 `AI로 상품 찾기`를 실행합니다.
5. 서버가 공개 상품 검색 결과를 수집하고, AI가 일정 맥락과 배송 정보 유무를 고려해 최대 3개 후보를 추천합니다.
6. 사용자는 상품을 선택하고 실제 구매가 끝나면 `구매 완료`로 표시합니다.
7. 오늘의 구매 탭은 마감일까지 남은 날짜에 따라 지금 챙길 항목을 정리합니다.

## 구현

- Frontend: React, Vite, JavaScript
- Backend: Node.js, Express, JavaScript
- AI: OpenAI Responses API의 JSON Schema 기반 구조화 출력
- Crawling: Cheerio를 이용한 네이버 쇼핑 공개 검색 결과의 제한적 수집
- Persistence: 브라우저 `localStorage`
- Deployment: Vite 빌드 결과를 Express가 정적 파일로 제공

브라우저에는 원본 메모, AI 제안, 확정한 구매 리스트, 구매 완료 기록, 선택한 상품 링크가 저장됩니다. 로그인은 필요하지 않습니다.

## 로컬 실행

Node.js 20 이상을 권장합니다.

```bash
npm install
copy .env.example .env
npm run dev
```

개발 중 프론트엔드는 `http://localhost:5173`, API 서버는 `http://localhost:3001`에서 실행됩니다. Vite 개발 서버가 `/api` 요청을 Express로 전달합니다.

API 키가 없어도 화면 흐름을 확인할 수 있습니다. 이 경우 예시 메모 분석과 수집된 상품 기준의 데모 추천을 사용하며, UI에 데모 모드임을 표시합니다.

## 환경 변수

`.env.example`을 `.env`로 복사하고 값을 설정합니다.

```dotenv
OPENAI_API_KEY=
OPENAI_MODEL=gpt-5-mini
PORT=3001
```

- `OPENAI_API_KEY`: 실제 AI 분석과 AI 상품 추천에 필요합니다.
- `OPENAI_MODEL`: Responses API와 구조화 출력을 지원하는 모델 이름입니다.
- `PORT`: Express 서버 포트입니다.

## 배포

```bash
npm install
npm run build
npm start
```

Express가 `dist` 폴더를 제공하므로 Node.js를 실행할 수 있는 서비스에 그대로 배포할 수 있습니다. 배포 환경에 `OPENAI_API_KEY`, `OPENAI_MODEL`, `PORT`를 설정해 주세요.

## 크롤링 방식

`server.js`의 `crawlNaverShopping()`은 사용자가 선택한 물품명을 검색어로 사용합니다. 공개 검색 HTML에서 상품 링크, 제목, 가격, 배송 관련 텍스트, 이미지 주소를 가능한 범위에서 추출합니다.

검색 페이지 구조가 바뀌거나 요청이 차단되면 상품별 정보를 얻지 못할 수 있습니다. 이때는 네이버 쇼핑 검색 결과 링크를 제공하고, 가격과 배송 정보를 확인해야 한다고 표시합니다. 배송 정보를 찾지 못한 경우에는 추측하지 않고 항상 `배송 정보: 확인할 수 없음`을 표시합니다.

## 설계 결정

- AI가 사용자의 보유 물품을 알 수 없으므로 `구매 필요`와 `확인 필요`로 임의 분류하지 않습니다.
- 직접 언급한 물품과 맥락상 제안하는 물품을 분리해 AI 추천의 근거를 투명하게 보여 줍니다.
- 구매 리스트에는 사용자가 확인해 담은 항목만 노출합니다.
- 상품 추천은 인기순만 따르지 않고 일정, 필요 날짜, 구매 마감일, 우선순위, 메모, 수집된 배송 정보 유무를 함께 고려합니다.
- 날짜가 불명확하면 AI가 ISO 날짜를 만들지 않고 `null`과 낮은 신뢰도를 반환하도록 지시합니다.

## MVP 한계

- 실제 캘린더 연동이 없습니다.
- 카카오톡이나 텔레그램 알림이 없습니다.
- 학교 안내문 자동 수집 기능이 없습니다.
- 결제 또는 장바구니 연동이 없습니다.
- 배송 예정일은 보장되지 않습니다.
- 크롤링 결과에 배송 정보가 없으면 `배송 정보: 확인할 수 없음`으로 표시합니다.
- 상품 추천은 수집된 제한적 검색 결과를 기반으로 하며 전체 시장을 비교하지 않습니다.
- 공개 검색 HTML 구조 변경이나 로봇 차단 정책에 따라 크롤링 결과가 달라질 수 있습니다.

## 향후 개선

- 가족별 일정과 캘린더 동기화
- 알림 채널 연동 및 구매 마감 리마인드
- 학교 안내문 이미지 및 PDF 분석
- 사용자별 선호 브랜드, 예산, 사이즈 저장
- 제휴 쇼핑 API를 이용한 안정적인 상품 데이터 수집
- 실제 배송 가능일 검증과 장바구니 연결
